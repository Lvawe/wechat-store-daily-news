# WeWe RSS 稳定性运维手册

> 针对「微信读书登录态间歇性失效导致 RSS 偶发抓不到内容」的问题，提供监控、提醒与恢复方案。
> 最后更新：2026-06-18

---

## 一、问题本质（先理解，再处理）

WeWe RSS 是**借微信读书的登录态**去拉取公众号历史文章的。这意味着：

- 微信读书的登录态会**间歇性失效**（被风控、登录态过期、微信读书侧波动等），失效期间某些公众号抓取会返回 **0 篇文章**，RSS 里就没有内容。
- 失效通常是**临时**的，很多时候过一会儿会自动恢复（实测 6/18 当天就出现过一次「抓不到 → 十几分钟后自动恢复」）。
- 这是 WeWe RSS 方案的**固有局限**，无法 100% 根除，只能通过「监控 + 自动兜底 + 提醒」把影响降到最低。

> 关键误区：feed 头部的 `<updated>` 时间是「该公众号源被添加的时间」，**不等于已经抓到文章**。判断有没有内容，要看 `<entry>` 标签的数量。

## 二、三层解决方案（已全部落地）

### 第 1 层：健康检查脚本（监控）

项目内已提供 `check_feeds.sh`，一条命令体检 5 个公众号：

```bash
bash /Users/lv/CodeBuddy/20260618102458/wechat-store-daily-news/wewe-rss/check_feeds.sh
```

它会自动：
1. 检查 wewe-rss 容器是否运行，没运行就自动 `docker compose up -d`；
2. 检查后台 4000 端口是否可访问；
3. 逐个列出 5 个公众号当前的 RSS 条目数；
4. 输出机器可读的结论标记：
   - `RESULT=HEALTHY` —— 全部正常；
   - `RESULT=DEGRADED` —— 有公众号为空（附 `EMPTY_FEEDS=` 列表），多为微信读书登录态临时失效；
   - `RESULT=DOWN` —— 容器/接口异常。

### 第 2 层：日报自动化任务的空源兜底（已写入 prompt）

「微信小店资讯日报」每天 10:00 运行时会：
1. **先跑健康检查脚本**；
2. 对**为空的公众号自动改用 web 搜索补内容**，并在该来源小节标注「⚠️ 本节 RSS 未取到内容，已改用 web 搜索补充」；
3. 在日报开头新增「**RSS 健康状态**」小节，如实写明哪个号没抓到、可能原因、处理建议。

这样即使某个号当天临时失效，**日报也不会漏内容、不会报错**，并且会清楚告诉你哪个号出了问题。

### 第 3 层：失效时的人工恢复流程

当健康检查或日报提示某个号长期为空（多次检查仍为 0），按下面顺序处理：

| 步骤 | 操作 |
|---|---|
| 1 | 打开后台 `http://localhost:4000`（登录码 `ZglDmcFElUzyPDo0xs6f`） |
| 2 | 进入「账号管理」，看微信读书账号状态 |
| 3 | 若显示**「失效」** → 重新扫码登录（别勾「24 小时自动退出」） |
| 4 | 若显示**「今日小黑屋」** → 被风控了，等 24 小时自动恢复，或重启容器：`docker compose restart` |
| 5 | 若账号正常但某号仍为 0 → 多为微信读书未收录该号，日报会自动走 web 搜索兜底，无需处理 |

## 四、降低失效频率的建议

1. **用微信读书小号绑定**：避免常用账号被频繁抓取触发风控。
2. **别勾「24 小时自动退出」**：扫码登录时取消勾选，否则每天都要重扫。
3. **保持 Docker 常开**：容器已配 `restart: unless-stopped`，Mac 重启后打开 Docker Desktop 即自动恢复。
4. **不要手动频繁刷新**：短时间内多次 `?update=true` 容易触发微信读书风控。
